1. WHAT IS THE .AGENT DIRECTORY?
================================================================================

The .agent directory is your CUSTOM AI WORKFLOW SYSTEM. It is a hand-crafted
framework that tells any AI assistant (like me) HOW to work on your project.

Think of it as a "playbook" or "instruction manual" for the AI.

It contains:
  - Agent role definitions (who does what)
  - An orchestrator (the master workflow/pipeline)
  - Memory files (project context, history, decisions)
  - Commands (reusable task blueprints)
  - Rules (always-on coding/SEO/UI standards)
  - Checklists (pre-deployment verification)
  - Logs (change history, daily logs, error tracking)
  - Prompts, Skills, Templets directories (currently empty, for future use)

3. THE ORCHESTRATOR (orchestrator.md) — THE BRAIN
================================================================================

This is the MOST IMPORTANT file. It defines TWO things:

A) STARTUP ROUTINE — What to do BEFORE answering any request:
   → Read all memory files (project, progress, client, features, bugs,
     decisions, sessions, stack)
   → Understand current state
   → Never ask for info that already exists
   → Continue from latest progress

B) 8-PHASE DEVELOPMENT PIPELINE — Work in this exact order:

   Phase 1: PROJECT MANAGER
     → Understand requirements, break into tasks, estimate, plan

   Phase 2: SOLUTION ARCHITECT
     → Choose architecture, folder structure, tech stack, database

   Phase 3: UI DESIGNER
     → Premium, mobile-first, accessible, professional design

   Phase 4: FRONTEND ENGINEER
     → Build reusable components, modular code, optimize

   Phase 5: BACKEND ENGINEER (only if needed)
     → Auth, database, storage, APIs, security

   Phase 6: SEO ENGINEER
     → Metadata, schema, OG tags, sitemap, robots.txt

   Phase 7: QA ENGINEER
     → Test responsiveness, forms, navigation, buttons,
       images, console, accessibility, performance

   Phase 8: DEPLOYMENT ENGINEER
     → Production build, deploy, env vars, verify, report

C) GENERAL RULES:
   → Never skip phases
   → Never jump directly into coding
   → Always review previous phase before next
   → Always produce production-ready work
   → Stop and ask if requirements incomplete

D) AFTER EVERY TASK:
   → Update progress.md, bugs.md, features.md, sessions.md, decisions.md


4. AGENT ROLES (agents/ directory)
================================================================================

4.1 PROJECT MANAGER (project-manager.md)
   - Break projects into phases
   - Create task lists
   - Estimate complexity
   - Prevent feature creep
   - Ask for missing requirements
   - Produce development roadmap
   - NEVER writes code
   - Always creates implementation plan first

4.2 SOLUTION ARCHITECT (solution-architect.md)
   - Choose technologies
   - Design folder structure
   - Choose database schema
   - Choose APIs
   - Design reusable architecture
   - Keep projects scalable
   - Never duplicate logic
   - Document important decisions

4.3 UI DESIGNER (ui-designer.md)
   - Premium UI
   - Modern layouts
   - Excellent typography
   - Balanced spacing
   - Beautiful gradients
   - Accessible colors
   - Glass effects only when appropriate
   - Every page must look agency quality

4.4 FRONTEND ENGINEER (frontend-engineer.md)
   - Uses: React, TypeScript, Tailwind
   - Reusable components
   - Never duplicate components
   - Never hardcode values
   - Always optimize performance
   - Always use semantic HTML

4.5 BACKEND ENGINEER (backend-engineer.md)
   - Preferred backend: Supabase
   - Handles: Authentication, Database, Storage, Policies, Edge Functions
   - Always secure APIs
   - Never expose keys

4.6 SEO ENGINEER (seo-engineer.md)
   - Currently EMPTY (no content defined)

4.7 QA ENGINEER (qa-engineer.md)
   - Tests: Forms, Links, Buttons, Navigation, Console,
     Performance, Accessibility
   - Tests on: Mobile, Tablet, Desktop
   - Never approves broken pages

4.8 DEPLOYMENT ENGINEER (deployment-engineer.md)
   - Handles: Production Build, Environment Variables,
     Netlify, GitHub, Performance, SEO Verification
   - Creates: Deployment Checklist, Post Deployment Report

================================================================================

1. STANDARDIZE how AI works on your projects — ensuring it always follows
   the same 8-phase pipeline (Plan → Architect → Design → Build Frontend

   the project, bugs, progress, decisions, or client info.

   and pre-deployment checklists.
